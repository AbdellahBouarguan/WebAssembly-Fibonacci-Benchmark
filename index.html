Theoretical Justification of Results

  1. JIT Warm-up & Interpreter Overhead: The V8 engine initiates JS execution via an interpreter (Ignition) before identifying 'hot' paths to compile via TurboFan. WebAssembly bypasses this phase, executing directly as a pre-optimized binary module.
  2. Static vs Dynamic Dispatch: The C implementation ensures strict static typing (int operations). Conversely, the JavaScript VM performs dynamic type checks and deoptimizations if numerical representations fluctuate between Smis (Small Integers) and HeapNumbers.
  3. Call Stack Efficiency: WebAssembly maintains a highly streamlined execution stack, allowing recursive frames to be pushed and popped with minimal memory overhead compared to full JS function contexts.
